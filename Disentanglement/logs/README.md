# Logs

See the top-level [`../README.md`](../README.md) for the project guide and
[`../EXPERIMENTS.md`](../EXPERIMENTS.md) for the indexed list of every run and
its result (auto-generated by `../analysis/build_index.py`).

## Live layout (current convention)

- `train/stage1/` — stage-1 SAE training `.out` logs.
- `train/stage2/<topic>/` — stage-2 training `.out` logs, grouped by topic
  (e.g. `grl_dense`, `routing_4exp`, `twoway_bothgn`, `scaled_postact`, …).
  Each `.out` opens with `=== <run>: <desc> ===  <date>`.
- `train/stage2/archive_failed_or_incomplete/` — kept for provenance, not clean results.
- `diag/<probe>/` — diagnostic probe `.out` logs (PR + SID stats-probe per bucket).

## Legacy (pre-reorg — superseded, do not add to)

- `probes/` — old probe logs; replaced by `diag/`. Anything labelled
  `*_old_lr_mislabeled_superb` used the underpowered SID probe and is **not**
  comparable to current stats-probe numbers.
- `figures/`, `analysis_figures/` — old figure outputs; current figures live in
  `../analysis/figures/`.

## Convention

Use exact folder names when reporting so diagnostic, legacy, and current
stats-probe evaluations are never mixed. Only `.out` (results) logs are kept —
`.err` stderr files are pruned during cleanup.
